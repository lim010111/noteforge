 Context                             

 docs/dev_contexts/v0.5_remains.md 의 두 항목:

 1. 배경 이미지 수동 결정: 현재 포스트 hero(배경)는 cover frontmatter(절대 URL//-anchored)가 있으면
 그것, 아니면 본문 첫 이미지로 자동 선택된다. "블로그 주인이 수동으로 결정"하고 "포스트에 배경 변경
 버튼"을 만들어야 한다.
 2. 리스트 썸네일: 홈/카테고리/폴더트리에서 날짜—제목으로만 표시되는 리스트가 밋밋하다. 포스트의 배경
 이미지를 기본 썸네일로 보여주고, 썸네일도 수동 변경할 수 있어야 한다.

 목표: 블로그 운영자가 (a) 포스트의 hero 배경을 본문 첫 이미지가 아닌 다른 이미지로 손쉽게 바꿀 수
 있고, (b) 리스트에 시각적 썸네일이 노출되며, 필요하면 hero 와 별개로 썸네일을 골라둘 수 있는 흐름을
 만든다.

 현재 인프라 상태 (조사 완료):
 - cover frontmatter 는 이미 allowlist + filterFrontmatter + loader heroImage 매핑까지 완비. 단 절대
 URL / /-anchored 경로만 허용.
 - Note.astro 는 note.heroImage 가 있으면 note-header__hero 배경 div 렌더링.
 - 본문에서 이미지 추출은 findFirstImageUrl() 한 개만 노출. 후보 이미지 리스트는 파이프라인에 노출 안
 됨.
 - 리스트는 세 곳 + TagPage: apps/blog/src/pages/index.astro (인라인),
 packages/theme-default/src/components/CategoryOverview.astro, .../FolderIndex.astro,
 .../NoteList.astro (TagPage).
 - 테마/앱 전반에 dev-only UI 패턴이나 frontmatter 역기록 엔드포인트는 아직 없음.
 - 중요: apps/blog/astro.config.mjs:16 output: 'static'. SSR API route 는 동작 안 함 → dev 전용 Vite
 middleware 를 astro-integration 에 둬야 함.
 - 중요: PipelineResult 에 slug → source path 매핑이 노출돼 있지 않음(notesBySlug 는 함수 내부 변수).
 Dev API 가 파일 경로를 알려면 sourcePathBySlug 를 신설 필요.

 Privacy 회귀 가드 (CRITICAL)

 ▎ 이 변경은 attachment URL 을 frontmatter 로 직접 받는 첫 사례. 무가드로 두면 우회 누출 경로.

 - Loader 게이트 강화: cover/thumbnail 값이 /attachments/<rel> 형태라면 <rel> 가
 result.attachmentClosure 의 멤버인지 검증, 아니면 무시. https:///http:// 절대 외부 URL 은 author 의
 명시적 선택이므로 그대로 통과.
 - 회귀 fixture: vault-mixed/ 에 private-only attachment 를 frontmatter cover/thumbnail 에 넣어둔
 케이스를 추가하고, 빌드 산출물 + heroImage/thumbnailImage 에 그 경로가 등장하지 않음을 검증. canary
 3종(DO_NOT_LEAK_BANANA_6f3c1, CLAUDE_COMMENT_LEAK_77b, FOLDER_TREE_DO_NOT_LEAK_8a4f2) 도 기존대로 0회
  검증.
 - Dev middleware 자체도 frontmatter 직접 갱신 통로이므로, 받은 cover/thumbnail 값에 동일 검증 적용 —
 closure 외 attachment 는 400 거부.
