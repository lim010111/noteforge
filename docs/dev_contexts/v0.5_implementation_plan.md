 확정된 결정 사항 (사용자 응답)

 - Q1 — 폰트 적용 범위: 본문+제목 모두 RIDIBatang. 한국어 글리프는 RIDIBatang, 영문은 기존

  폴백 자리를 RIDIBatang으로 일괄 교체.

 없으면 본문 첫 이미지로.

 - Q4 — RIDIBatang 자산: 사용자 직접 제공. 구현 단계에서 apps/blog/public/fonts/RIDIBatang.woff2 +
  라이선스 파일이 존재한다고 가정.

 T3: Attachment 파이프라인 — 이미지 표시 복구

 진단 (확정):
 - packages/core/src/pipeline.ts:339 result.attachmentClosure: Set<string> — vault-relative 경로
 (예: attachments/AI/.../foo.jpg)
 - packages/core/src/pipeline.ts:207 attachmentUrlFor: (id) => '/attachments/${id}' — HTML에
 박히는 URL은 /attachments/<vault-rel-path>
 - packages/astro-integration/src/loader.ts — attachmentClosure를 읽지도, 복사하지도 않는다.
 dist에 attachment가 들어갈 통로가 없음.
 - packages/cli/src/commands/audit.ts:54 등에서 closure를 검증하지만, 검증 시점엔 이미 파일이
 dist에 있어야 함을 전제 → 누락된 단계.

 해결 — 두 경로 모두 처리:

 1. 빌드(astro:build:done): vault에서 dist로 복사
   - packages/astro-integration/src/integration.ts L143-147 placeholder 훅을 실제 동작으로 교체
   - 옵션 A — integration 훅에서 runCorePipeline(config) 재호출하여 closure 획득. 비용은 캐시 +
 최후 단계 1회라 허용 가능.
   - 옵션 B — loader가 closure를 module-scope에 저장 → integration이 읽기 (의존성 정합 더 깔끔).
 선호.
   - 복사 로직: 각 vault-rel-path를 dir/attachments/<vault-rel-path>로 fs.copyFile (디렉토리 mkdir
  -p). dir은 hook 인자의 dir URL.
   - 복사 대상은 attachmentClosure만 (allAttachments가 아님 — 비공개로만 참조된 자산을 dist로
 흘리지 않는 privacy 계약 준수)
 2. 개발(astro:server:setup): vite middleware로 /attachments/*를 vault에서 직접 서빙
   - server.middlewares.use('/attachments', ...)로 closure에 들어 있는 경로만 통과시키고
 외부는 404. closure 외 경로 접근은 leak이므로 거부.
   - 메모리(project_obsidian_publish_oss)의 privacy-first 원칙: 어떤 우회도 closure 멤버십 검사를
 거쳐야 함.
 3. 표준 마크다운 ![](url) 분기:
   - 절대 URL(http(s)://)는 통과(브라우저가 직접 fetch).
   - 상대 경로는 wikilink 정규화 단계에서 attachment basename 매칭으로 흡수되거나, 매칭 실패 시
 그대로 broken (현 상태). 이번 PR은 wikilink 경로만 보장하고, 임의 상대경로 path는 별도 이슈로
 분리. 필요시 사용자에게 실제 노트의 마크다운 형태 확인 요청.

 privacy 회귀 방어:
 - packages/core/tests/fixtures/vault-mixed/의 canary(DO_NOT_LEAK_BANANA_6f3c1,
 CLAUDE_COMMENT_LEAK_77b, FOLDER_TREE_DO_NOT_LEAK_8a4f2) 외에 private 폴더 내부 attachment가
 dist에 절대 복사되지 않음을 검증하는 테스트가 있는지 확인 필요. 없으면 attachment용 canary
 픽스처(예: private/_secret_attach.png) 추가하여 build 후 dist에 등장 0회 단언.
