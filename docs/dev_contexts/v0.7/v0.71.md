현재 사이드바에 Categories가 있잖아. 거기에서 Depth가 2인 구조와 Depth가 1인 구조에서 UI 차이가 좀 있거든?  
  
예를 들어서 Depth가 2인 것은 'PEFT', 그다음 'LoRA'. 이렇게 Depth가 2인데, Depth가 1인 구조는 'news'라고 가정하자. Depth가 달라도 'PEFT와' 'news'는 같은 레벨이기 때문에 이렇게 딱 보기 좋게 정렬이 돼 있어야 되는데, 'PEFT'는 하위 Depth가 하나 더 있기 때문에 그 열고 닫기가 가능하잖아.  
  
그것 때문에 지금 UI가 깨지는 것 같아. 같은 레벨에서는 이게 딱 정렬된 상태로 보여야 되는데, 그렇게 보일 수 있게 만들어.  
  
그리고 Categories는 지금 gaps만, 그러니까 Parents Directory, Parents Directory 즉 Parents Category를 보여주고 있잖아. 근데 이 카테고리 이름만 보여주니까 그 안에 구조가 어떻게 되고 몇 개의 문서가 있을지 User 입장에서 불편할 수 있어.  
  
카테고리명 오른쪽에 숫자로 몇 개의 문서가 들어가 있는지, 이 카테고리에 몇 개가 문서가 들어가 있는지 숫자로 나타낼 수 있게 해.
  
예를 들어 'PEFT'는 LoRA라는 하위 child, child parents가 있지만 어쨌든 'PEFT'라는 parents는 "LoRA 란"이라는 하나의 문서를 가지고 있으니까 `1`.  
  
그리고 'LoRA'라는 parents도 "LoRA 란"이라는 문서를 가지고 있으니까 옆에 `1` 이렇게 문서의 개수를 보여줄 수 있게 그렇게 개발을 해.
